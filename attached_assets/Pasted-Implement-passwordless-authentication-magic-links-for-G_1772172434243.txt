Implement passwordless authentication (magic links) for Ghost Job Detector.

Constraints:
- Do NOT implement username/password.
- Users sign in by email -> receive a magic link -> clicking link logs them in.
- Backend is Node/Express TypeScript.
- Browser extension calls POST /api/analyze; it must require auth.
- Keep friction low; after sign-in once, extension works via session cookies.

Requirements:
1) Database
   - Create users table (id, email, createdAt, lastLoginAt)
   - Create login_tokens table (id, userId/email, tokenHash, expiresAt, usedAt, createdAt, ip, userAgent)
   - Token must be random, single-use, expires in 15 minutes.

2) Backend routes
   - POST /api/auth/request-link
     - body: { email }
     - Always return 200 { ok: true } (avoid account enumeration)
     - Create token, hash it, store with expiry, send magic link via email provider.
   - GET /api/auth/magic?token=...
     - Validate token (exists, not used, not expired)
     - Mark used, create/find user by email
     - Create session cookie (httpOnly, secure in production, sameSite compatible)
     - Redirect to /app (or a simple “You’re signed in” page)
   - POST /api/auth/logout
   - GET /api/auth/me returns { authenticated: true, email } or { authenticated: false }

3) Protect endpoints
   - Require authenticated session for:
     - POST /api/analyze
     - /api/analyses*
     - any /api/ai/*
   - On 401 return:
     { error: "Unauthorized", action: "login_required", loginUrl: "<BASE_URL>/login" }

4) CORS + cookies for extension
   - Explicitly allow origin: chrome-extension://<EXTENSION_ID> and the app domain
   - Set Access-Control-Allow-Credentials: true
   - Ensure extension fetch uses credentials: "include"

5) Email sending
   - Use a transactional email provider (Resend or SendGrid preferred).
   - Read API key from env (RESEND_API_KEY or SENDGRID_API_KEY).
   - From address from env (MAIL_FROM).
   - Subject: "Your Ghost Job Detector sign-in link"
   - Email body includes the magic link and states it expires in 15 minutes.

6) Frontend
   - Create /login page with email input + “Send link”
   - After submit, show “Check your email” state
   - Create /app page that confirms signed-in status

7) Extension UX
   - When /api/analyze returns login_required:
     - Show panel message “Sign in to unlock AI analysis”
     - Button “Sign in” opens loginUrl in new tab (chrome.tabs.create)
   - After user signs in, next click works (no auto-run).

8) README updates
   - Setup env vars: RESEND_API_KEY, MAIL_FROM, BASE_URL
   - How to test locally and deployed
   - Security notes: no password storage, tokens hashed, single use, expiry
