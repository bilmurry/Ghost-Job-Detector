Implement passwordless authentication using magic links, optimized for a Chrome extension flow.

Goals:
- No username/password.
- User enters email -> receives magic link -> click logs in.
- After clicking the magic link, show a simple confirmation page:
  "You're signed in. Return to your job tab and click the ghost."
- Protect AI endpoints so only authenticated users can call them.
- Extension must work via session cookies with credentials.

Backend Requirements:
1) Add DB tables:
   - users: id, email (unique), createdAt, lastLoginAt
   - login_tokens: id, email, tokenHash, expiresAt, usedAt, createdAt, ip, userAgent
   Token rules:
   - random token (>= 32 bytes)
   - store only hash (sha256)
   - expires in 15 minutes
   - single-use (usedAt)

2) Add routes:
   - POST /api/auth/request-link
     body { email }
     Always return 200 { ok: true }
     Create token, store hash + metadata, email a link:
       BASE_URL + "/api/auth/magic?token=..."
   - GET /api/auth/magic?token=...
     Validate (exists, not expired, not used)
     Mark usedAt
     Create/find user by email
     Create authenticated session cookie
     Redirect to "/signed-in"
   - GET /api/auth/me -> { authenticated: true, email } else { authenticated: false }
   - POST /api/auth/logout

3) Add a simple public UI page:
   - GET /signed-in returns HTML:
     - big “Signed in ✅”
     - message: “Return to your job tab and click the ghost again.”
     - optional button: “Open Job Sites” linking to LinkedIn/Indeed

4) Protect endpoints:
   - Require authenticated session for:
     POST /api/analyze
     GET/POST /api/analyses*
     any /api/ai/*
   - If unauthorized, return:
     401 { error:"Unauthorized", action:"login_required", loginUrl: BASE_URL + "/login" }

5) CORS + cookies:
   - No wildcard origins.
   - Allow:
     - the deployed app origin (BASE_URL)
     - chrome-extension://<EXTENSION_ID>
   - Set credentials true.
   - Ensure cookies are httpOnly, secure in production, sameSite compatible for extension use.
   - trust proxy enabled if needed for secure cookies.

6) Extension changes:
   - All fetch calls to backend must use:
     credentials: "include"
   - If response is 401 with action=login_required:
     - show UI: “Sign in to unlock AI analysis”
     - button opens loginUrl in a new tab
     - do not auto-run after login

Email Provider:
- Use Resend preferred.
- Env vars:
  RESEND_API_KEY
  MAIL_FROM
  BASE_URL
- Email content:
  subject: "Your sign-in link"
  body: link + "expires in 15 minutes"

Deliver:
- Exact file changes
- Test steps:
  - call /api/auth/request-link
  - click email link -> land on /signed-in
  - /api/analyze returns 401 before login and succeeds after
